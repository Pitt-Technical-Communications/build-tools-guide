Getting Started

In general, programmers use desktop or laptop systems to develop software. To use this guide, the reader will need access to a desktop or laptop system with a well-supported Windows, macOS, or Linux operating system. Depending on which operating system you use, this page includes separate instructions for Windows, Mac, and Linux users.

Windows Users

If you want to write software on a Windows system, there are two options that this guide recommends. If you are using Windows 10 version 2004 or above or Windows 11, see the Windows Subsystem for Linux (WSL) section. Otherwise, go to the Installing MinGW section.

Windows Subsystem for Linux (WSL)

The Windows Subsystem for Linux (WSL) enables developers to use Linux tools on a Windows system. This is an all-in-one software package containing all the tools necessary in a build system. Please see Microsoft's tutorial for installing WSL.

Installing MinGW

MinGW (Minimalist GNU for Windows) is an open source build environment for building Windows programs. It includes the GNU Compiler Collection (GCC), GNU Build Utils (assembler, linker, archive manager), GNU Debugger (GDB), and Windows libraries. MinGW produces 32-bit executables that can run on 32-bit or 64-bit versions of Windows. To build 64-bit executables, there is a separate build environment called MinGW-w64, but that is outside the scope of this guide. Regardless, much of the process between these two environments is identical.
